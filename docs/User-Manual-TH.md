# คู่มือการใช้งานระบบ PowerCare.CM

คู่มือนี้จัดทำขึ้นสำหรับการใช้งานระบบ `PowerCare.CM` ในงาน Corrective Maintenance (CM) ของโรงไฟฟ้า เพื่อให้ผู้ใช้งานแต่ละบทบาทสามารถทำงานในระบบได้อย่างถูกต้องและต่อเนื่อง

## 1. วัตถุประสงค์ของระบบ

ระบบนี้ใช้สำหรับ:

- รับแจ้งซ่อมจากผู้แจ้งซ่อมโดยไม่ต้อง login
- ติดตามสถานะงานซ่อมด้วยเลขที่ CM
- ให้ช่างและวิศวกรรับงาน ดำเนินงาน ตรวจรับ และปิดงาน
- ให้ผู้ดูแลระบบจัดการผู้ใช้งาน ข้อมูลหลังบ้าน และประวัติการเปลี่ยนแปลง
- พิมพ์เอกสารปิดงานเมื่อสถานะงานปิดแล้ว
- ส่งออกข้อมูลรายงานเป็น Excel

## 2. กลุ่มผู้ใช้งาน

ระบบแบ่งผู้ใช้งานหลักออกเป็น 4 กลุ่ม:

### 2.1 ผู้แจ้งซ่อม

- ไม่ต้องมี username หรือ password
- สามารถแจ้งซ่อมได้ทันที
- สามารถติดตามสถานะงานด้วยเลขที่แจ้งซ่อม
- ไม่สามารถพิมพ์เอกสารปิดงาน

### 2.2 ช่าง (Technician)

- รับงานในหมวดที่ตรงกับตนเอง
- เริ่มดำเนินการ
- บันทึกผลการซ่อม
- ส่งงานรอตรวจรับ/ปิดงาน
- ปล่อยงานกลับเข้าคิวได้หากรับผิดงานหรือยังทำต่อไม่ได้
- อัปโหลดลายเซ็นและรูปโปรไฟล์ในหน้า Profile

### 2.3 วิศวกร (Engineer)

- รับงานในหมวดของตนเองได้
- ตรวจทานงานที่ช่างส่งมา
- ส่งกลับให้แก้ไขได้
- ปิดงานได้
- ยกเลิกงานได้ในหมวดที่ตนเองรับผิดชอบ
- ส่งออก Excel ได้
- อัปโหลดลายเซ็นและรูปโปรไฟล์ในหน้า Profile

### 2.4 ผู้ดูแลระบบ (Admin)

- ดูได้ทุกงาน ทุกหมวด
- สร้าง แก้ไข รีเซ็ตรหัสผ่าน และลบผู้ใช้งาน
- เพิ่ม/ปิดใช้งาน Category และ Zone
- ปรับ SLA
- ดู History และ Audit Trail
- ยกเลิกงานได้ทุกงาน
- ส่งออก Excel ได้

## 3. เมนูหลักของระบบ

เมื่อ login แล้ว เมนูหลักทางซ้ายประกอบด้วย:

- `Dashboard`
- `All Work`
- `Reports`
- `Profile`
- `Create Request`
- `Track Work`

ถ้าเป็น `Admin` จะมีเพิ่ม:

- `Admin`
- `History`

ด้านบนของระบบมี:

- ปุ่ม `Home` กลับหน้า Dashboard
- ปุ่ม `Day/Night`
- ปุ่ม `Logout`

## 4. ความหมายของสถานะงาน

สถานะหลักที่ใช้ในระบบมีดังนี้:

- `แจ้งใหม่` : งานเพิ่งถูกสร้างเข้าระบบ
- `รอรับงาน` : งานรอให้ผู้มีสิทธิ์ในหมวดเดียวกันมารับงาน
- `รับเรื่องแล้ว` : มีผู้รับงานแล้ว
- `กำลังดำเนินการ` : ช่างกำลังทำงาน
- `รอปิดงาน` : ช่างทำเสร็จและส่งให้วิศวกรตรวจรับ
- `ส่งกลับให้แก้ไข` : วิศวกรส่งงานกลับให้ช่างแก้ไขหรือเติมข้อมูล
- `ปิดงานแล้ว` : งานเสร็จสมบูรณ์และพิมพ์เอกสารได้
- `ยกเลิก` : งานถูกยกเลิกและไม่ดำเนินการต่อ

## 5. การใช้งานสำหรับผู้แจ้งซ่อม

### 5.1 แจ้งซ่อม

ไปที่หน้า `Create Request` หรือปุ่ม `แจ้งซ่อมทันที` จากหน้าแรก

กรอกข้อมูลต่อไปนี้:

- ชื่อผู้แจ้ง
- หน่วยงาน/แผนก
- Category
- Zone
- ชื่อเครื่องจักร
- หัวข้อปัญหา
- รายละเอียดปัญหา
- ระดับความเร่งด่วน

จากนั้นกด `ส่งแจ้งซ่อม`

เมื่อส่งสำเร็จ ระบบจะแสดง:

- ข้อความส่งแจ้งซ่อมสำเร็จ
- เลขที่แจ้งซ่อม
- ปุ่มไปหน้าติดตามสถานะ

สิ่งสำคัญ:

- ต้องจดหรือบันทึกเลขที่แจ้งซ่อมไว้
- เลขนี้ใช้สำหรับติดตามงานในภายหลัง

### 5.2 ติดตามสถานะงาน

ไปที่หน้า `Track Work`

กรอกเลขที่แจ้งซ่อม เช่น:

`CM-2026-06-0001`

ระบบจะแสดง:

- สถานะล่าสุด
- Category
- Zone
- ความเร่งด่วน
- ผู้รับงาน
- รายละเอียดงาน
- Timeline การเปลี่ยนสถานะ

หากไม่พบเลขที่แจ้งซ่อม ระบบจะแจ้งว่าไม่พบข้อมูล

## 6. การใช้งานสำหรับช่าง

### 6.1 Login เข้าระบบ

ไปที่หน้า `Staff Login`

กรอก:

- Username
- Password

เมื่อเข้าระบบแล้วให้เริ่มที่หน้า `Dashboard` หรือ `All Work`

### 6.2 ดูภาพรวมงาน

หน้า `Dashboard` ใช้ดู:

- จำนวนงานทั้งหมด
- งานแจ้งใหม่
- งานที่อยู่ระหว่างดำเนินการ
- งานปิดแล้ว
- งานยกเลิก
- กราฟสถานะ
- แนวโน้มรายเดือน
- ภาระงานตาม Zone
- งานที่ควรรีบดำเนินการ

สามารถกรองได้ตาม:

- Overview
- Mechanical
- Electrical
- ช่วงเวลา

### 6.3 เปิดรายการงานทั้งหมด

ไปที่ `All Work`

สามารถค้นหาและกรองได้ตาม:

- เลข CM / ชื่อเครื่องจักร / ชื่อผู้แจ้ง
- สถานะ
- หมวดงาน
- Zone
- ความเร่งด่วน
- ผู้รับงาน
- เดือน

หน้ารายการงานแสดงสูงสุด 50 รายการต่อหน้า และมีปุ่มเปลี่ยนหน้า

### 6.4 รับงาน

ช่างสามารถรับงานได้เฉพาะหมวดที่ตรงกับตนเอง

ตัวอย่าง:

- ช่างไฟฟ้ารับงานไฟฟ้าได้
- ช่างเครื่องกลรับงานเครื่องกลได้
- มองเห็นงานต่างหมวดได้ แต่รับไม่ได้

ขั้นตอน:

1. เปิดหน้า `All Work` หรือหน้ารายละเอียดงาน
2. กด `รับงาน`
3. ระบบจะเปลี่ยนสถานะไปตามขั้นตอนการทำงาน

### 6.5 เริ่มดำเนินการ

เมื่อรับงานแล้ว ให้เปิดหน้ารายละเอียดงานและกด `เริ่มดำเนินการ`

### 6.6 บันทึกผลการซ่อม

เมื่อดำเนินการแล้ว ให้กรอก:

- สาเหตุ
- วิธีการแก้ไข
- หมายเหตุ

จากนั้นกด `ส่งรอปิดงาน`

### 6.7 ปล่อยงานคืนคิว

หากรับงานผิด หรือยังทำต่อไม่ได้ สามารถปล่อยงานคืนคิวได้

ต้องกรอก:

- เหตุผลปล่อยคืนคิว

หลังจากนั้นงานจะกลับไปรอรับงานอีกครั้ง

## 7. การใช้งานสำหรับวิศวกร

### 7.1 รับงาน

วิศวกรสามารถรับงานในหมวดของตนเองได้เช่นกัน

### 7.2 ตรวจรับงาน

เมื่อช่างส่งงานมาในสถานะ `รอปิดงาน` หรือ `ส่งกลับให้แก้ไข`

วิศวกรสามารถ:

- อ่านรายละเอียดงาน
- ตรวจทานข้อมูลการซ่อม
- ดูสาเหตุและวิธีการแก้ไข
- ใส่หมายเหตุวิศวกร

### 7.3 ปิดงาน

หากข้อมูลครบและงานเสร็จจริง:

1. เปิดหน้ารายละเอียดงาน
2. กรอกหมายเหตุวิศวกร หากมี
3. กด `ปิดงาน`

เมื่อปิดงานแล้ว:

- สถานะจะเปลี่ยนเป็น `ปิดงานแล้ว`
- สามารถพิมพ์เอกสารปิดงานได้
- ระบบจะดึงลายเซ็นจากโปรไฟล์ของช่างและวิศวกรอัตโนมัติ

### 7.4 ส่งกลับให้แก้ไข

หากข้อมูลไม่ครบหรือยังต้องแก้ไข:

1. กรอกเหตุผล
2. กด `ส่งกลับให้แก้ไข`

งานจะถูกส่งกลับให้ช่างดำเนินการเพิ่มเติม

### 7.5 ยกเลิกงาน

วิศวกรสามารถยกเลิกงานในหมวดที่ตนเองรับผิดชอบได้ โดยต้องระบุเหตุผลทุกครั้ง

## 8. การพิมพ์เอกสารปิดงาน

เอกสารปิดงานจะพิมพ์ได้เมื่อ:

- งานอยู่ในสถานะ `ปิดงานแล้ว`
- ผู้ใช้งาน login เข้าระบบแล้ว

ขั้นตอน:

1. เปิดหน้ารายละเอียดงาน
2. กด `พิมพ์เอกสาร`

เอกสารจะแสดงข้อมูลสำคัญ เช่น:

- เลขที่แจ้งซ่อม
- ผู้แจ้ง
- หน่วยงาน
- Category
- Zone
- เครื่องจักร
- รายละเอียดปัญหา
- วันที่ปิดงาน
- รายละเอียดการแก้ไข
- ลายเซ็นของผู้ดำเนินการและผู้ตรวจรับ

## 9. การใช้งานสำหรับ Admin

### 9.1 จัดการผู้ใช้งาน

ไปที่หน้า `Admin`

Admin สามารถ:

- สร้างผู้ใช้งานใหม่
- แก้ไข Username
- แก้ไขชื่อ-นามสกุล
- แก้ไขหน่วยงาน
- เปลี่ยน Role
- เปลี่ยน Category
- Reset password
- เปิด/ปิดการใช้งานบัญชี
- อัปโหลดลายเซ็นแทนผู้ใช้
- อัปโหลดรูปโปรไฟล์แทนผู้ใช้

### 9.2 ลบผู้ใช้งาน

เมื่อต้องการลบผู้ใช้งาน:

1. กดปุ่มลบ
2. ระบบจะแสดงกล่องยืนยัน
3. ต้องยืนยันด้วยรหัสผ่าน Admin

ถ้าถูกต้อง:

- ระบบลบผู้ใช้สำเร็จ
- แสดงผลสำเร็จ
- บันทึกลงประวัติ

ถ้ารหัสผ่านไม่ถูกต้อง:

- ระบบจะแจ้งว่าไม่สำเร็จ

### 9.3 จัดการ Category

หน้า `Categories` ใช้สำหรับ:

- เพิ่ม Category ใหม่
- ปิดใช้งาน Category ที่ไม่ต้องการใช้แล้ว

### 9.4 จัดการ Zone

หน้า `Zones` ใช้สำหรับ:

- เพิ่ม Zone ใหม่
- ปิดใช้งาน Zone

### 9.5 ตั้งค่า SLA

หน้า `SLA Settings` ใช้กำหนดเกณฑ์งานค้าง เช่น:

- ค้างรับงาน
- ค้างดำเนินการ
- ค้างตรวจปิด

### 9.6 ดูประวัติการเพิ่ม/ลบ

หน้า `History` ใช้ตรวจสอบ:

- การเพิ่ม User
- การลบ User
- การเพิ่มใบแจ้งซ่อม
- การเพิ่มหรือปิดใช้งาน Category
- การเพิ่มหรือปิดใช้งาน Zone

### 9.7 ดู Audit Trail

หน้า `Audit Trail` ใช้ตรวจสอบการกระทำสำคัญในระบบ เช่น:

- รับงาน
- เริ่มดำเนินการ
- ส่งรอตรวจรับ
- ส่งกลับให้แก้ไข
- ปิดงาน
- ยกเลิกงาน
- การแก้ไขข้อมูลผู้ใช้

## 10. การจัดการรูปโปรไฟล์และลายเซ็น

ไปที่หน้า `Profile`

### 10.1 รูปโปรไฟล์

ใช้สำหรับ:

- แสดงตัวตนในหน้า Work Results
- แสดงในหน้า Admin Users

ข้อแนะนำ:

- ใช้รูปหน้าตรง เห็นชัด
- ไฟล์ไม่ใหญ่เกินที่ระบบกำหนด

### 10.2 ลายเซ็น

ใช้สำหรับ:

- แสดงในเอกสารปิดงาน

ข้อสำคัญ:

- ช่างและวิศวกรควรอัปโหลดลายเซ็นก่อนใช้งานจริง
- เมื่อปิดงาน ระบบจะดึงลายเซ็นจากโปรไฟล์อัตโนมัติ
- ไม่ต้องอัปโหลดใหม่ทุกครั้งที่ปิดงาน

## 11. การส่งออกข้อมูลรายงาน

ไปที่หน้า `Reports`

สิทธิ์ใช้งาน:

- Admin ใช้ได้
- Engineer ใช้ได้
- Technician ใช้ไม่ได้

ขั้นตอน:

1. เปิดหน้า `Reports`
2. กด `Export Excel`

## 12. ข้อควรระวังในการใช้งาน

- ผู้แจ้งซ่อมต้องเก็บเลขที่แจ้งซ่อมทุกครั้ง
- ช่างควรตรวจหมวดงานก่อนรับงาน
- วิศวกรควรตรวจความครบถ้วนก่อนปิดงาน
- Admin ควรตรวจสอบก่อนลบผู้ใช้ เพราะเป็นการลบจริง
- ควรอัปโหลดรูปโปรไฟล์และลายเซ็นให้ครบก่อนเริ่มใช้งานจริง

## 13. ขั้นตอนแนะนำในการใช้งานจริงประจำวัน

### ช่วงเริ่มกะ

1. Login เข้าระบบ
2. เปิดหน้า Dashboard
3. ดูงานใหม่และงานค้าง
4. เปิด All Work เพื่อกรองงานของตนเอง

### ระหว่างวัน

1. รับงานตามหมวดที่รับผิดชอบ
2. อัปเดตสถานะเมื่อเริ่มดำเนินการ
3. บันทึกผลเมื่อทำงานเสร็จ
4. ส่งรอปิดงาน

### ก่อนจบวัน

1. ตรวจงานที่ยังค้าง
2. ตรวจงานที่รอตรวจรับ
3. พิมพ์เอกสารสำหรับงานที่ปิดแล้ว หากจำเป็น

## 14. คำถามที่พบบ่อย

### ถาม: ผู้แจ้งซ่อมต้องมี account หรือไม่

ตอบ: ไม่ต้อง สามารถแจ้งซ่อมและติดตามงานได้โดยไม่ต้อง login

### ถาม: ถ้าลืมเลขที่แจ้งซ่อมจะทำอย่างไร

ตอบ: ต้องให้ผู้เกี่ยวข้องค้นจากหน้า `All Work` หรือให้ Admin ช่วยตรวจสอบข้อมูลย้อนหลัง

### ถาม: ช่างรับงานข้ามหมวดได้หรือไม่

ตอบ: ไม่ได้ ช่างรับได้เฉพาะหมวดที่ตรงกับตนเอง

### ถาม: วิศวกรต้องอัปโหลดลายเซ็นทุกครั้งตอนปิดงานหรือไม่

ตอบ: ไม่ต้อง ระบบดึงจากโปรไฟล์อัตโนมัติ

### ถาม: งานยกเลิกสามารถพิมพ์เอกสารปิดงานได้หรือไม่

ตอบ: ไม่ได้

## 15. ข้อแนะนำเพิ่มเติม

หากต้องการใช้งานในองค์กรอย่างเป็นระบบ แนะนำให้มีเอกสารประกอบเพิ่มเติมอีก 2 ชุด:

- คู่มือฉบับย่อ 1 หน้า สำหรับผู้ใช้งานหน้างาน
- คู่มือสำหรับ Admin โดยเฉพาะ สำหรับการจัดการผู้ใช้และตรวจสอบย้อนหลัง

---

เอกสารฉบับนี้เหมาะสำหรับใช้งานเป็นคู่มือเริ่มต้น, ใช้ประกอบการอบรม, และใช้เป็นเอกสารอ้างอิงระหว่างเริ่มใช้งานระบบจริง
